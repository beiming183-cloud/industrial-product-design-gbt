# Security Policy

## Reporting a vulnerability

Do not publish credentials, private project data, unsafe executable payloads, or a reproducible exploit in a public issue. Contact the repository owner privately through their GitHub profile and include the affected commit, component, impact, and a minimal reproduction.

## Product-safety findings

Product design, electrical safety, compliance, and certification findings are not automatically software vulnerabilities. Open a design-review issue only when disclosure is safe and the example contains no customer data, proprietary geometry, restricted standards content, or instructions that create immediate physical risk.

This project does not provide emergency support, certification, or a warranty of product safety.
