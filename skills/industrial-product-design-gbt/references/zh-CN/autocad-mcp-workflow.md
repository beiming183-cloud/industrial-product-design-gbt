# AutoCAD MCP 工作流

通过 MCP、文件 IPC、COM、脚本或其他自动化桥操作 AutoCAD 时，必须读取本参考。

工作流应与具体桥实现无关。`system.ensure_ready`、事务操作和 JSON 载荷只是所需能力或推荐契约的名称；只有当前桥实际暴露时才可调用，否则使用可用的安全等价方式并记录限制。

## 目录

- 就绪与能力发现
- 失败契约与有限恢复
- 严格请求和后置条件契约
- 桥一致性测试
- 机械制图设置
- 事务与实体归属
- 几何与标注执行
- 二维/三维能力和回退约定
- 几何 DRC
- 预览、打印与比例证明
- 交付与 DXF 重导入
- 机械基元与齿轮
- 标准工作流

## 就绪与能力发现

把就绪视为幂等状态迁移，而不是状态查询的隐藏副作用。

1. 从环境或桥发现已安装 AutoCAD 产品和可执行文件路径，不得硬编码 `AutoCAD LT`、版本或用户专属路径。
2. 检查 AutoCAD 是否运行并识别实际产品/版本。
3. 等待活动文档；只有用户要求新图时才创建文档。
4. 通过版本握手和 IPC ping 验证调度器/桥已加载、兼容且响应正常。
5. 记录 AutoCAD 产品/构建、调度器/桥/后端版本、传输、协议/API schema、启用工具集/能力清单及相关请求/响应 schema。升级或版本不一致后，先运行无害只读探针，再信任可变调用。
6. 桥暴露 `system.ensure_ready` 类操作时优先使用；否则仅以已暴露的安全工具依次执行发现/启动/等待/加载/握手/ping 等价步骤。
7. `status` 只负责观察，不得在只读外观的调用中隐藏启动、LISP 加载、创建文档或重试。

分别返回或记录安装、进程、文档、调度器版本、传输和就绪状态。不得从窗口缺失或调度器错误推断 AutoCAD 产品版本。

采用有限恢复：

- 对未变化的失败状态，自动启动或重载可信调度器最多一次。
- 恢复动作后重新 ping。
- 重试未改变状态时停止并报告精确失败状态，不能循环启动或重试 IPC 超时。
- 同一文档或文件 IPC 队列上的可变 CAD 调用不得并行执行。

## 失败契约与有限恢复

只要载荷中包含错误，就按失败处理，即使外层 MCP 包装错误地报告 `isError: false`。判断成功前规范化嵌套文本 JSON。

每次可变操作后，在发出依赖操作前检查命令完成状态、文档状态、事务结果、创建/修改句柄及重建/约束警告。表面成功但修剪结果为空、连接失败、约束未解、批次部分完成或文档陈旧，都不算成功。

比较请求目标/数值与操作后实体以及变更句柄/几何指纹。`ok` 但缺少要求的语义变化、出现非预期自动调整、坐标/状态串写或目标集外修改，均为硬失败。幂等“确保状态”操作只有在前态已满足完整请求，且响应以回读证据明确报告 `already_satisfied` 时，才可无修改通过；其他无操作结果都要停止后续修改，直至回滚阶段并解决原因。

失败响应应优先采用结构化形式：

```json
{
  "ok": false,
  "error": {
    "code": "E_DISPATCHER_NOT_LOADED",
    "message": "AutoCAD is running but the dispatcher is unavailable",
    "recoverable": true,
    "recommended_action": "autoload_dispatcher"
  }
}
```

桥提供足够证据时，至少区分：

- `E_AUTOCAD_NOT_INSTALLED`
- `E_AUTOCAD_NOT_RUNNING`
- `E_NO_ACTIVE_DOCUMENT`
- `E_DISPATCHER_NOT_LOADED`
- `E_DISPATCHER_VERSION_MISMATCH`
- `E_IPC_TIMEOUT`
- `E_COMMAND_STATE_BLOCKED`
- `E_OUTPUT_PATH_REJECTED`
- `E_POSTCONDITION_MISMATCH`

存在错误码或状态探针时，不得从消息片段猜测恢复动作。桥缺陷与图纸缺陷分开报告。

## 严格请求和后置条件契约

优先使用不可变、强类型、自包含的请求对象。每个可变请求必须显式携带全部必需坐标/参数，绝不允许从全局或上一命令状态继承省略的 `x1`、`y1`、`x2`、`y2`、图层、样式或几何字段。桥支持时采用 `strict: true` 或等价 schema 模式，在修改前拒绝缺失、类型错误或未知多余字段。

为每项操作分配唯一请求/阶段 ID。响应必须返回：

- `requested`：完成显式单位转换后的规范化请求值。
- `actual`：创建/更新后从 AutoCAD 回读的实际值。
- `diff`：带公差和单位的字段级数值/语义差异。
- 所有创建/修改句柄、文档/路径身份、事务状态及警告/错误状态。

立即查询每个创建/修改句柄。比较实体类型、图层、`component_id`、`line_class`、坐标/端点/顶点、半径/角度、文字内容/边界、闭合状态、关联关系和操作拥有的其他参数。优先使用原生语义元数据；实体无法安全存储时，在操作清单/旁证文件中保存同样字段。

任何无法解释的不一致都是 `E_POSTCONDITION_MISMATCH`，不是 `WARNING`：

1. 停止全部依赖绘图调用。
2. 回滚原子批次；没有事务时，仅在获得授权后删除已跟踪的新实体，并证明清理完成。
3. 重置/重载桥一次，然后运行最小一致性探针。
4. 同一故障再次出现时，切换到已验证结构化后端或把工作标为 `blocked`。禁止继续修复或装饰语义坐标不可信的实体。

文档和打印操作的后置条件还包括实际活动文件名/路径、介质、可打印区域、打印模式、计算比例和输出身份。请求了文件名但活动文档仍为 `Drawing1.dwg`，或请求固定比例却实际自适应打印，都属于后置条件失败。

## 桥一致性测试

桥/后端/schema 更新后，以及怀疑坐标串写或后置条件不一致时运行一致性测试。正常绘图任务不必每次运行完整压力测试。

- 用唯一哨兵坐标/内容交错创建 `LINE`、`RECTANGLE`、`POLYLINE` 和 `MTEXT`，至少运行 100 个确定性循环；回读所有句柄，证明请求或批次间没有字段泄漏。
- 在严格模式下组合缺失、多余和类型错误字段，证明无效请求会在不创建实体的情况下被拒绝。
- 注入一次批次失败，证明原子回滚后实体数量、句柄和几何指纹不变。
- 重复测试图形创建/命名、保存/导出和打印，并比较请求与实际文档/输出身份。
- 记录桥/后端/schema 版本、种子/输入夹具、每项操作的 `requested`/`actual`/`diff`、失败索引、清理结果和最终图形指纹。

一致性夹具失败时，修复并通过夹具或选择已验证后端前，阻断该后端的全部可变操作。

## 机械制图设置

创建几何前，先建立或核对一个明确设置约定：

```json
{
  "standard": "GB/T",
  "units": "mm",
  "sheet": "A3",
  "orientation": "landscape",
  "projection": "first-angle",
  "scale": "1:1"
}
```

图样受 GB/T 管理，且用户、源文件和批准模板均未指定投影法时，把第一角画法作为已记录的默认值。不能根据 AutoCAD 地区设置推断投影法；应同时检查视图配置、投影符号和标题栏字段。

把约定应用到真实 CAD 状态，而不只写入交付元数据：

- 图形单位和插入单位。
- 图层名称、线型和线宽。
- 文字、尺寸、引线和表格式样。
- 图纸空间布局、页面设置、打印设备、纸张、可打印区域、方向、打印样式、打印范围和比例模式。
- 图框/标题栏属性，其图幅、投影法、单位和比例应与打印设置一致。
- 完整打印和编辑所需的外部参照、块定义/属性、字体/SHX、打印样式、页面设置、图像、数据链接、自定义/代理对象和支持路径。

中文文字应依据适用 GB/T 字体要求及打印产物验证。`gbenor` 配 `gbcbig` 或已安装 TrueType 替代字体，只有在检查汉字覆盖、技术字体比例、缺失/替换符号、文字边界和 PDF 输出后才算有效。

优先使用白名单变量设置操作，限制变量名、类型和范围，设置后回读。不得只为修改单位或尺寸变量而绕过桥执行任意 LISP。

尺寸文字高度等数值是按图幅/样式选择的配置，不是通用常数。

## 事务与实体归属

优先采用原子批量创建或显式 `begin` / `commit` / `rollback` 语义。

- `continue_on_error: false` 只表示停止后续调用，不能证明已经回滚。
- 桥支持时要求 `atomic: true` 或 `rollback_on_error: true`。
- 没有事务时，在唯一暂存图层/块上创建新内容，记录每个返回句柄，验证批次后再提升。失败时只有获得删除授权才移除已跟踪的新实体。
- 不得用大范围删除或图层清理在用户图纸中临时模拟回滚。
- 爆炸半径不确定时保留原图并保存到新修订/副本。

维护包含源操作 ID、实体句柄、图层、角色和分组的操作清单，用于定向修复和证据报告。

## 几何与标注执行

- 支持时使用 AutoCAD 原生实体和关联尺寸。
- 重复特征优先使用块、阵列、约束或参数，而不是复制松散几何。
- 创建响应必须返回所有相关句柄。引线加文字操作应分别标识引线、文字和关联/组，不能只返回顶层实体类型。
- 对标注变更分组，保证可共同移动、删除或回滚。
- 使用原生标注样式、明确字高和碰撞/间隙检查。自动放置后重新审计；只有视觉复核通过，自动避让才成为证据。
- API 缺少原生或关联对象时，记录回退方式及其可编辑性/导出限制。

## 二维/三维能力和回退约定

选择构建方法前发现能力，并记录活动桥是否安全支持：

- 二维编辑：端点/对象捕捉、`trim`、`extend`、`break`、`join`/多段线编辑、圆角、相切构建、块、阵列和关联尺寸。
- 约束：重合、相切、同心、等半径、水平/垂直、对称和尺寸约束。
- 三维建模：基本实体、拉伸、旋转、扫掠、适用时放样、并集/差集/交集、圆角、倒角、阵列、装配变换、剖切和投影视图。
- 可靠性：原子事务/回滚、完整实体查询与关联、确定性保存/导出，以及支持覆盖或内容哈希证据的非缓存预览。

选择工具和证据能够支持的最高表达层级。安全三维操作或投影视图不可用时，采用共同参数化二维骨架，并披露结果只是二维概念/教学表达。不得用分别目测估算的前视图、侧视图冒充三维权威来源。

AutoCAD 桥重复出现 `E_POSTCONDITION_MISMATCH` 时，在可用条件下优先采用 `ezdxf` 等确定性结构化 DXF 生成器/解析器：在故障修改路径外生成参数化几何，再让 AutoCAD/MCP 仅执行打开、回读、审计、已证明安全的标注和打印。没有已验证后端时，停止并标记 `blocked`。

每个可变批次只包含一个完整子系统或可复核阶段。骨架及每个子系统完成后，先审计句柄、拓扑、共享参数、受影响视图和新预览，再继续。批次返回成功只能证明调用返回，不能证明修剪、连接、相切、遮挡或装配正确。

清理、换层、拓扑修复及其他宽范围修改在桥支持时优先采用 dry-run/preview。执行前把拟修改句柄集与子系统清单比较。

## 几何 DRC

表达复核前先运行变更范围几何 DRC。Tier 3 发布/导出工作还要对每个适用的导出/重导入产物再次运行；Tier 1 不得只为重复检查而重新生成未受影响的交换文件。

直线、圆弧、圆和多段线至少检查：

- 连续重复顶点和重复闭合顶点。
- 零长度和低于公差的线段。
- 零/无效半径、非有限坐标和退化圆弧。
- 应闭合却开放的轮廓。
- 自交、局部反向、尖刺，以及方向有意义时的不一致绕向。
- 重复实体或反向重复实体。
- 共线重叠线段和非预期重合几何。
- 悬空端点、应连接却近接未接的端点、非预期交叉、未修剪遮挡几何、突出线段和必需相切断裂。
- 依据共享参数检查等径/同心组、重复特征节距和投影位置。
- 异常实体孤岛、图层/类型不一致以及会打印的散落构造几何。

使用带单位的明确模型空间公差。每项失败必须给出实体句柄、线段/子实体索引、位置、实测值和阈值。实体数、闭合标志、保存成功和 DXF 可读不能证明几何干净。

同时按语义角色和几何审计。区分组件轮廓、材料边界、中心/隐藏/构造线、尺寸/引线、剖面线、表格/标题栏线和有意开放端口，不能让表格或标注几何稀释组件拓扑发现。

优先修复源生成器并重新生成。自动清理只能在新生成/已跟踪几何上移除连续重复顶点或零长度段，且必须保持拓扑和意图不变。清理后重新运行全部受影响审计。

## 预览、打印与比例证明

严格区分预览和发布打印：

- PNG 预览操作必须以明确分辨率/DPI 生成 PNG，或返回图像内容/缩略图数据。
- PDF 打印操作必须按明确页面设置生成打印 PDF。
- 只有 PDF 时，使用可用结构化 PDF 渲染器转成位图并披露回退；不得把 PDF 路径伪称为 PNG 预览。
- 使用唯一输出名或显式覆盖，并验证内容哈希或修改身份，避免把缓存预览误认成当前图形。
- 迭代时保持相机/视图、图层可见性、背景、分辨率和比例稳定，以产生有意义的前后视觉差异。无差异可能表示预览陈旧；无关差异表示出现非预期修改或渲染状态不稳定。
- 视觉复核前检查图像尺寸和非空白/非均匀像素。只请求风险所需的元数据、缩略图和全分辨率视图，避免把重复未变化图像塞入 Agent 上下文。

每次发布打印必须显式设置并验证：

- 打印设备及介质/纸张名。
- 方向和可打印区域。
- 布局/窗口/范围打印区域。
- 单色或要求的打印样式。
- 固定比例或自适应纸张。
- 居中/偏移和线宽行为。

标题栏为 `1:1` 时必须固定按 1:1 打印。记录计算比例，并拒绝误标为 1:1 的自适应结果。重新打开 PDF，在全图和可读细节视图中检查空白、裁切、重叠、字体/符号缺失、线宽错误及比例/标题栏不一致。

`PLOT_SCALE_CONSISTENCY` 报告必须包含请求模式/比例、实际介质/打印区域/变换/计算比例、标题栏声明和 PDF 证据。采用自适应时不得显示 `1:1` 等数字工程比例；应使用批准的非比例标识（如 `NTS`），并记录打印模式 `FIT`。

## 交付与 DXF 重导入

从已验证的权威图形生成交付物，然后重导入或独立解析交换产物。

除非用户明确要求 DXF 作为活动源，否则把 DXF 视为导出物。执行 `save_as_dxf` 类操作后，验证活动文档和权威 DWG/原生源未静默切换为 DXF。

对 DXF 记录并比较：

- `$INSUNITS`、`$ACADVER`、坐标精度和范围。
- 按类型和图层分组的实体数。
- 按声明公差规范化的几何指纹。
- 多段线顶点数、闭合标志、凸度/圆弧数据和线段长度。
- 圆/圆弧中心、半径和角度。
- Unicode 文字和属性。
- 原生尺寸/引线保留情况，或是否炸开为线、文字、代理对象。
- 关键尺寸和参考坐标。

把源图、DXF 重导入、打印 PDF 和清单视为不同证据渠道。总实体数相同只是一项粗略清单检查。

最终报告必须说明实际文件、纸张、打印比例、单位、DRC 结果、重导入比较、假设、不支持对象和每个 `NOT_EVALUATED` 项。

正式发布应采用 `product-definition-release.md`：打包所需 Xref/字体/打印样式/模板或把它们报告为外部依赖，并把每个产物绑定到准确图形修订/配置及生成它的 AutoCAD/桥版本。

实体查询必须返回修复和证明所需参数，包括句柄、端点、圆弧角度/端点、多段线顶点/凸度、MText 宽度/边界、块成员、组、约束和关联关系。桥缺失必需字段时，把受影响检查标为 `NOT_EVALUATED`。

## 机械基元与齿轮

桥提供领域基元时使用它们。不得把普通多段线宣传为已验证机械齿轮。

明确分类齿轮输出：

- `symbolic`：仅为图解符号。
- `simplified`：用于布局/沟通的近似齿形，标记 `NOT FOR MANUFACTURING`。
- `manufacturing`：具有完整设计输入的真实、已验证渐开线几何。

直齿、内齿和行星齿轮按需验证：

- 模数、压力角、齿数、变位、齿顶/齿根和中心距。
- 根切风险；无变位标准全齿高外齿轮在声称标准齿形前，应把齿数与适用最小值（如 `2 / sin(alpha)^2`）比较。
- 内外啮合兼容、干涉、重合度、侧隙和齿厚。
- 标准同心行星布置满足 `Zr = Zs + 2*Zp`。
- 等间隔行星装配相位，适用布置中检查 `(Zs + Zr) / Np` 的整除条件。
- 行星轮互相间隙、行星架销位、齿相位以及完整运动周期内的运动/干涉。

制造输出必须具有真实渐开线和工艺/公差数据。桥缺少基元时，用经过验证的机械库计算；否则保持符号/简化表达并披露限制。

## 标准工作流

运行序列前按 `cad-workflows.md` 选择验证层级。Tier 1 局部修改可在完成定向修改、重算、变更句柄/几何证明、依赖检查和一次受影响视图检查后停止；不得只为填满清单而处理未变更的打印/导出/重导入步骤。Tier 3 发布使用完整序列，仅跳过真正不适用的产物。

```text
发现能力和安装
-> 确保 AutoCAD/调度器/文档就绪
-> 创建或核对机械制图设置
-> 开始原子/暂存事务
-> 创建结构化几何与标注
-> 审计几何
-> 审计表达
-> 打印并检查预览
-> 提交/提升事务
-> 交付 DWG/DXF/PDF
-> 重导入并比较 DXF 几何
-> 发布产物和证据
```

不得把“几何审计”“表达审计”“打印复核”和“交换格式重导入”合并成一个泛化的 `validated` 标志。
